# 莱莎客户聊天记录复盘分析 (Chat Analysis)

本技能对业务员与客户之间的完整聊天记录进行系统化复盘，从动作拆解、话术评估、策略匹配、结果归因四个维度输出结构化分析报告。目标是让每一次沟通都成为可复用的销售资产。

---

## 1. 分析前置：信息采集与校验

### 1.1 必需信息

分析启动前，必须确认以下信息已提供或可推断：

| 信息项 | 来源 | 缺失处理 |
|--------|------|----------|
| **完整聊天记录** | 用户直接提供 / WhatsApp 导出 / 阿里TM截图 | 缺记录 = 无法分析，须要求用户提供 |
| **客户名称** | 用户告知 / 背调记录 | 缺名称 = 无法关联 wiki，标记为"匿名客户" |
| **客户国家** | 用户告知 / wiki 客户页 | 缺国家 = 无法匹配市场策略 |
| **客户类型** | wiki `buyer_personas.md` 五大客群 | 缺类型 = 分析前先判断，明确标注推断依据 |
| **沟通渠道** | WhatsApp / 阿里TM / Email / Instagram | 缺渠道 = 无法评估渠道合规性 |
| **沟通时间段** | 记录中的时间戳 | 缺时间 = 无法评估节奏与响应速度 |
| **最终结果** | 成交 / 未成交 / 谈判中 / 流失 | 缺结果 = 标记为"状态未知"，但不影响过程分析 |

### 1.2 背景补充（主动执行）

分析前，按以下顺序补充客户背景：

1. 读取 `wiki/clients/` 中对应客户文件（如有），提取 MCPC 画像与历史交互记录
2. 读取 `wiki/strategies/buyer_personas.md`，判断客户所属五大客群之一
3. 读取 `wiki/strategies/English_USPs.md` 与 `wiki/tech/product_grades.md`，建立产品话术基准线
4. 读取 `wiki/strategies/Price_Objection.md` 与 `wiki/strategies/Shipping_Speed.md`，建立策略基准线
5. 若客户文件不存在，标记"无历史背调"并在报告中建议补充

## 2. 聊天记录分段与动作拆解

### 2.1 分段原则

将聊天记录按"沟通回合"分段，一个回合 = 客户发一条 + 业务员回应（含后续补充）。分段后为每个回合编号：R1, R2, R3...

### 2.2 动作标签体系

对每个回合中业务员的行为，标注以下动作标签（可叠加）：

| 动作标签 | 定义 | 正向示例 | 负向示例 |
|----------|------|----------|----------|
| **开场破冰** | 首次接触建立关系基调 | 自我介绍+问句结尾 | 直接推销无铺垫 |
| **需求探询** | 主动了解客户痛点/需求 | "What's your biggest challenge with..." | 跳过需求直接报价 |
| **产品展示** | 展示产品特色/USP | 引用 Genius Weft 0.5mm 特性 | 泛泛说"质量好" |
| **证据背书** | 用客观数据/案例佐证 | "90% reorder rate" / 水洗测试视频 | 纯主观判断无依据 |
| **报价策略** | 报价/议价/谈判 | B1/B3 分级报价+利润结构 | 报价无逻辑，随意降价 |
| **异议处理** | 回应客户反对意见 | FABE 结构回应价格异议 | 回避或妥协 |
| **推进闭环** | 引导客户做出决策/下单 | "Would you like to start with a sample pack?" | 消息无方向，等客户主动 |
| **情感共鸣** | 与客户建立情感连接 | "I understand how frustrating that can be" | 机械式回复无温度 |
| **信息确认** | 确认关键细节（地址/规格/数量） | "Could you confirm the exact length you need?" | 假设客户意图不确认 |
| **节奏把控** | 控制回复速度与频率 | 客户犹豫时给 24h 空间 | 连续轰炸式追发 |
| **防流失锚定** | 流失预警时进行挽回 | "Before you decide, let me share one thing..." | 客户沉默就放弃 |
| **合规守线** | 避免触碰平台红线 | 阿里TM 中不提线下支付 | 发 WhatsApp 号引导线下 |

### 2.3 客户动作标签（对应标注）

| 动作标签 | 定义 |
|----------|------|
| **兴趣表达** | 主动询问产品/价格/流程 |
| **异议提出** | 表达反对/质疑/犹豫 |
| **沉默/拖延** | 不回复或用模糊话拖延 |
| **决策信号** | 明确下单/确认规格/付定金 |
| **流失信号** | "I'll think about it" / 不再回复 / 找到别家 |

## 3. 六维度评估模型

对完整聊天记录进行以下六个维度的量化评估（1-5 分制）：

### 3.1 响应时效 (Response Speed)

| 分数 | 标准 |
|------|------|
| 5 | 24h 内全覆盖响应，关键回合 <2h |
| 4 | 24h 内响应，但关键回合偶有延迟 |
| 3 | 大部分 24h 内，偶有超 24h |
| 2 | 经常超 24h，影响谈判节奏 |
| 1 | 多次超 48h，客户体验严重受损 |

**关联策略**：wiki `compliance_system.md` 中阿里 24h 响应率红线；wiki `Shipping_Speed.md` 中 12h 发货承诺的呼应逻辑。

### 3.2 话术专业度 (Talk Track Quality)

| 分数 | 标准 |
|------|------|
| 5 | 系统引用 USP（0.5mm / 低温慢漂 / 保鳞），FABE 结构完整，问句结尾 |
| 4 | 引用部分 USP，结构较完整，偶有问句结尾 |
| 3 | 有产品知识但不系统，缺少结构化话术 |
| 2 | 泛泛描述，无差异化卖点 |
| 1 | 话术错误或误导（如混淆 B1/B3） |

**关联策略**：wiki `English_USPs.md` 全部产品卖点；wiki `product_grades.md` B1/B3 差异化；wiki `Price_Objection.md` FABE 模板。

### 3.3 需求匹配度 (Needs Alignment)

| 分数 | 标准 |
|------|------|
| 5 | 客户类型精准判断（沙龙/品牌/电商/批发/影视），推荐产品完全匹配痛点 |
| 4 | 判断较准确，推荐产品基本匹配 |
| 3 | 判断模糊，推荐产品方向对但细节偏 |
| 2 | 推荐不匹配客户需求 |
| 1 | 完全不了解客户需求，盲目推销 |

**关联策略**：wiki `buyer_personas.md` 五大客群痛点与匹配策略。

### 3.4 谈判推进力 (Deal Momentum)

| 分数 | 标准 |
|------|------|
| 5 | 每个回合都有推进方向，闭环意识强，3-5 回合内达成样品/小单 |
| 4 | 推进较主动，但偶有回合方向不明 |
| 3 | 推进力度不足，多回合原地踏步 |
| 2 | 完全被动，等客户驱动 |
| 1 | 无推进逻辑，聊天变成闲聊 |

**关联策略**：whatsapp-hair-sales skill 的三段式跟进序列（T+3/T+7/T+14）；wiki `order_moq_policy.md` 样品单 20% 费用与返还机制。

### 3.5 情感连接度 (Emotional Engagement)

| 分数 | 标准 |
|------|------|
| 5 | 建立个人连接，客户主动分享业务故事/痛点 |
| 4 | 有情感共鸣，客户愿意透露部分信息 |
| 3 | 礼貌但机械，缺乏个人连接 |
| 2 | 纯交易式沟通 |
| 1 | 冷硬或过度热情（刷屏） |

**关联策略**：whatsapp-hair-sales skill 中 CEO 级 Bullet Points vs 发型师级情感共鸣话术区分。

### 3.6 合规与风控 (Compliance & Risk)

| 分数 | 标准 |
|------|------|
| 5 | 全程合规，阿里 TM 无线下支付关键词，客户背调信息有标注置信度 |
| 4 | 基本合规，偶有模糊边界表述 |
| 3 | 有潜在合规风险但未越线 |
| 2 | 明确违规（如阿里 TM 发线下支付关键词） |
| 1 | 严重违规或触发风控 |

**关联策略**：wiki `compliance_system.md` 客户资产保护与阿里红线；whatsapp-hair-sales skill 中背调置信度三级体系。

## 4. 关键话术提取

### 4.1 提取规则

从聊天记录中提取以下两类话术：

**亮点话术（Golden Phrases）** — 值得全团队复用的高质量表达：
- 必须是业务员的原话（或经过微调的版本）
- 必须具备可复用性（去掉客户特定信息后仍可适用）
- 标注适用场景（开场/异议处理/推进闭环/情感共鸣）

**问题话术（Red Flags）** — 需要修正的低质量表达：
- 标注具体问题类型（被动/模糊/误导/越线）
- 给出修正建议（基于 wiki 话术模板）

### 4.2 话术评估对照表

提取话术后，逐条对照以下 wiki 资源评估质量：

| 评估维度 | 对照资源 | 检查点 |
|----------|----------|--------|
| 产品描述准确性 | `wiki/tech/product_grades.md` | B1/B3 特性是否混用 |
| USP 引用完整性 | `wiki/strategies/English_USPs.md` | 是否遗漏核心卖点 |
| 价格异议回应质量 | `wiki/strategies/Price_Objection.md` | 是否使用 FABE 结构 |
| 发货承诺准确性 | `wiki/strategies/Shipping_Speed.md` | 12h/24h 承诺是否一致 |
| MOQ/样品政策准确性 | `wiki/strategies/order_moq_policy.md` | 样品费 20% 是否提及 |
| 9610 报关描述准确性 | `wiki/strategies/9610_customs_clearance.md` | 报关流程是否准确 |
| 渠道合规性 | `wiki/strategies/compliance_system.md` | 阿里红线是否触碰 |
| 客户类型话术匹配 | `wiki/strategies/buyer_personas.md` | 品牌版 vs 沙龙版是否正确选用 |

## 5. 结果归因分析

### 5.1 成交案例归因

若聊天记录最终成交，分析成交的 **核心驱动因素**（不多于 3 个）：

| 驱动因素 | 判断依据 |
|----------|----------|
| 产品差异化优势 | 客户明确提及认可某 USP（如 0.5mm / 低温慢漂） |
| 价格策略得当 | 客户对报价接受度高，异议处理顺畅 |
| 供应链响应速度 | 客户关注交期，12h/24h 发货成为决策推动力 |
| 情感连接与信任 | 客户主动分享业务信息，表现出信任 |
| 样品策略闭环 | 样品体验后直接转大单 |

### 5.2 未成交/流失案例归因

若未成交，分析流失的 **根因**（不多于 3 个）：

| 流失根因 | 判断依据 |
|----------|----------|
| 价格鸿沟 | 客户预算远低于莱莎定价区间，异议处理后仍无法跨越 |
| 需求错配 | 推荐的产品线/等级与客户实际需求不匹配 |
| 节奏失控 | 回复延迟导致客户转向竞品 |
| 推进力不足 | 业务员未主动推进闭环，客户沉默后流失 |
| 信任缺失 | 客户始终不透露真实需求/预算 |
| 竞品截胡 | 客户明确提及已找其他供应商 |
| 合规障碍 | 阿里平台限制或 9610 报关流程阻碍 |

## 6. 优化建议与行动计划

### 6.1 建议分层

所有优化建议按优先级标注：

| 优先级 | 标记 | 适用场景 |
|--------|------|----------|
| 紧急 | 🔴 | 合规违规 / 客户已流失且可挽回 / 话术严重错误 |
| 重要 | 🟡 | 谈判推进力不足 / 话术专业度偏低 / 需求匹配偏差 |
| 建议 | 🟢 | 情感连接可提升 / 响应时效微优化 / 话术模板微调 |

### 6.2 建议必须包含

每条建议必须包含：
1. **问题描述**：一句话说明发现了什么问题
2. **影响评估**：这个问题导致了什么后果（客户反应/流失风险/成交概率下降）
3. **修正方案**：具体的可执行修正步骤（引用 wiki 话术模板页 / SOP 文件名）
4. **预期收益**：修正后预计带来的改善

### 6.3 下一步行动清单

报告末尾必须输出"下一步行动清单"，格式：

```
### 下一步行动清单
1. [🔴] 立即修正：[具体动作] → [负责人] → [截止日期]
2. [🟡] 本周完成：[具体动作] → [负责人]
3. [🟢] 持续优化：[具体动作] → [负责人]
```

## 8. 特殊场景处理

### 8.1 WhatsApp 渠道专项

若沟通渠道为 WhatsApp，额外评估：

| 评估项 | 标准 | 对照资源 |
|--------|------|----------|
| 单条字数控制 | ≤50 词/条（极简气泡消息） | whatsapp-hair-sales §3 |
| 图片+文字节奏 | 严禁连续 2+ 图片不带文字 | whatsapp-hair-sales §3 |
| 模拟人工节奏 | 不短时间密集发多条模板消息 | whatsapp-hair-sales §3 |
| 话术对等匹配 | CEO→Bullet Points / 发型师→情感共鸣 | whatsapp-hair-sales §3 |
| 三段式跟进序列 | T+3/T+7/T+14 是否遵循 | whatsapp-hair-sales §2 |

### 8.2 阿里 TM 渠道专项

若沟通渠道为阿里 TM，额外评估：

| 评估项 | 标准 | 对照资源 |
|--------|------|----------|
| 线下支付关键词 | 严禁发送任何线下支付引导词 | wiki `compliance_system.md` §3 |
| 24h 响应率 | 全覆盖响应 | wiki `compliance_system.md` §3 |
| 首回模板质量 | 是否按品牌版/沙龙版区分 | leshine-sales-sop `03_talk_tracks/` |

### 8.3 多轮长对话（>15 回合）

超过 15 回合的对话，回合分析改为 **阶段式汇总**：

- **阶段 1：破冰与需求确认**（R1-R5）：整体节奏、需求挖掘深度
- **阶段 2：产品展示与异议处理**（R6-R10）：USP 覆盖率、异议回应质量
- **阶段 3：报价与谈判推进**（R11-R15）：报价逻辑、推进力度
- **阶段 4：决策与闭环**（R16+）：闭环意识、客户决策信号捕捉

每个阶段给出阶段评分（1-5）和阶段简评。

## 10. 调用方式与分发规则

### 10.1 TL 分发规则

本技能属于 TL 强制分发范围 — 当业务员提交聊天记录要求复盘分析时，TL 不得自行执行全流程分析，应分发给 @Message Crafter 或 @Client Scout 协作完成。

但以下情况 TL 可直接读取本技能执行：
- 业务员仅提交少量回合（≤5 回合）的简短复盘请求
- 业务员要求快速评估一条话术的质量

### 10.2 多 Agent 协作场景

对于复杂案例（成交大单复盘 / 流失大客户归因 / 新人完整培训复盘），可调动多个 Agent：

| 协作场景 | 涉及 Agent | 各方职责 |
|----------|-----------|----------|
| 成交大单复盘 | @Client Scout + @Message Crafter | Scout 补充客户背调深度；Crafter 评估话术质量 |
| 流失客户归因 | @Client Scout + @Market Radar | Scout 补充竞品截胡证据；Radar 分析市场价格鸿沟 |
| 新人培训复盘 | @Message Crafter + TL | Crafter 提取话术问题清单；TL 汇总培训建议 |
