# 培训速递（Training Digest）需求与实现方案

> 2026-07-17 反向访谈定稿。参考形态：https://aihot.virxact.com/ 的日报式速览发布。

## 一、问题本质（第一性原理）

培训知识有两个衰减点：

1. **参训人脑中的一手体感**——24 小时内衰减大半，且每次参训的人都不同，无法靠个人习惯保证输出质量；
2. **同事的注意力**——同事需要的不是原始资料，而是「**与我有关的、可执行的增量**」：这场培训讲了什么新东西、对我的岗位有什么用、第一步怎么做。

所以系统要做的事只有一件：**在衰减发生前，把「一手体感 + 原始资料」压缩成结构化、可行动的速览，推到同事面前**。原始资料只作为可下载的附录，不是主角。

## 二、访谈共识（方向性决策）

| 决策点 | 结论 | 理由 |
|---|---|---|
| 发布载体 | 方舟平台新领域模块 `app/training/` | 复用登录/RBAC/钉钉推送/附件下载全套基建；同事本来就在平台上；可沉淀可检索 |
| 发布人 | 参训同事自助发布，**流程强引导** | 参训人每次不同，不能卡在任何单人身上；质量下限由模板 + AI 起草 + 必填校验保证，不靠自觉 |
| 覆盖范围 | V1 只做培训模板，表结构留 `type` 扩展位 | 模板专为培训设计引导性才强；将来扩展会/展会/复盘时加模板即可，不改架构 |

**按最合理假设处理的点**（未访谈，可推翻）：

- 录音/视频转写**不进 V1**：录音用钉钉 AI 听记转文字后，把文字稿贴进材料框即可（公司在钉钉生态内，现成能力）；
- **无审核流**：发布即生效，`training:admin` 可下架/修改任何速览；
- **无评论区**：讨论发生在钉钉群里（推送卡片就是讨论锚点），平台侧只做轻量「有用」反馈。

## 三、核心链路（5 步）

```
① 采集          ② AI 提炼          ③ 校对发布        ④ 分发            ⑤ 消费
培训一结束，     多模态解析材料，    AI 草稿逐区预填，   发布即推钉钉群     AI HOT 式详情页
向导式创建，     按培训模板生成      参训人只「改」      actionCard        + 列表/标签/搜索
丢材料即可       结构化草稿 JSON     不「写」；必填      （标题+3 条重点    + 原始资料下载
（手机可用）                        分区不过不能发       +链接）           + 「有用」反馈
```

### 强引导的落地方式（本方案的灵魂，对应亮哥的核心要求）

创建流程是**四步向导（stepper）**，不是一张大表单：

1. **基本信息**：培训名称、日期、机构/讲师、参训人（默认当前用户，可多选补充）、标签（平台类：TikTok/亚马逊/独立站…；主题类：投流/选品/AI 工具…，字典管理）。能自动带出的不让用户填。
2. **丢材料**：拖拽多文件（PPT/PDF/图片）+ 粘贴文字（笔记/听记转写/群聊摘录），点「AI 提炼」。等待期间显示进度反馈，不是转圈干等。
3. **逐区校对**：AI 草稿按模板分区预填，参训人逐区修改。每区有明确的格式要求和示例占位文本。
4. **发布预览**：整页预览 + 「发布并推送钉钉群」。

**模板分区（固定，必填项标 ★）**：

| 分区 | 要求 | 由谁产出 |
|---|---|---|
| ★ 一句话总结 | ≤50 字，这场培训到底讲了什么 | AI 起草，人校对 |
| ★ 重点 Top 3~5 | 每条一句话 + 一段展开 | AI 起草，人校对 |
| 亮点 / 新知 | 与我们已有认知的**差异点**，不是复述大纲 | AI 起草，人筛选 |
| ★ 可应用点 | 每条必须选「适用岗位/业务」+ 写「落地第一步」 | AI 给候选，**人拍板** |
| 方法与技巧 | 可操作的步骤/口诀/工具用法 | AI 起草，人校对 |
| ★ 参训人点评 | 值不值、哪部分水、建议谁重点看——**AI 不代写**，前端不预填 | 人 |
| 原始资料 | 附件列表，下载即得 | 系统 |

**校验规则**：★ 分区未达标（空、低于条数/字数下限）时发布按钮禁用，并明确指出缺什么（反馈引导行动，不是弹「校验失败」）。AI 起草兜底保证：哪怕只丢几张 PPT 照片，参训人面对的也是「改草稿」而不是「写作文」——把门槛从写作降到校对，这是自助发布能成立的前提。

### 消费侧（对齐 AI HOT 形态）

| AI HOT | 培训速递 |
|---|---|
| 日报详情页：元信息头（日期/篇数/阅读时长）+ 锚点目录 + 分区条目 + 来源 | 速览详情页：培训名/日期/讲师/参训人/预计阅读时长 + 锚点目录 + 分区卡片 + 底部资料下载区 |
| 精选流 | 速递列表页（时间倒序，卡片含一句话总结 + 标签） |
| 主题聚合 | 标签筛选（平台/主题）+ **按「适用岗位」筛选**（解决「跟我有关」） |
| 收藏 | V1 用「有用 👍」代替（计数 + 防重复） |
| RSS 订阅 | 钉钉群推送（企业语境里钉钉就是 RSS） |

移动场景是硬要求：培训常在外地结束，「第一时间」大概率发生在手机上。按约定 21 走**主站响应式**，编辑器四步向导必须在手机上可完整走通。

## 四、技术方案

### 数据模型（1 个迁移，2~3 张表）

- `training_digest`：`id` / `type`(默认 'training'，扩展位) / `title` / `org` / `lecturer` / `trained_at` / `attendees`(JSON) / `tags`(JSON) / `summary` / `sections`(JSON，结构化分区内容) / `status`(draft/published) / `view_count` / `useful_count` / `created_by` / `published_at`
  - `sections` 用 JSON 而非每区一列：将来加模板不用迁移；★ 必填校验在 service 层按模板 schema 做。
- `training_digest_file`：`digest_id` / `file_name` / `file_path`(锚定 REPO_ROOT，cerebrum 2026-07-03 教训) / `file_size` / `mime` / `uploaded_by`
- `training_digest_feedback`：`digest_id` / `user_id` / `kind`('useful')，唯一约束防重复；浏览数直接落主表计数。

### 后端 `app/training/`（四件套）

- `router.py`：CRUD、附件上传/下载（JWT 鉴权即可，钉钉卡片点进来是主站页面自带登录态）、`POST /draft`(AI 起草)、`POST /publish`、`POST /useful`。全部端点挂 `require_permission` + `ok()` 信封。
- `service.py` 拆三个子 service：
  - `digest_service`：CRUD + draft→published 状态机 + 必填 schema 校验；
  - `draft_service`：材料 → `app.ai.service.chat`（多模态，preset 后台可配）→ 严格 JSON 草稿（schema 校验失败自动重试一次）。PDF/PPT 先抽文本，图片直接走多模态 image_url（不传 detail，参数用 max_tokens——约定 7）；
  - `push_service`：复用 dingtalk 模块发群机器人 actionCard（标题 + 一句话总结 + 3 条重点 + 详情链接）。实施前查 `docs/module-notes.md` 确认现有钉钉能力口径。
- 权限：`seed_role_permissions` 加 `training:read`（全员）/ `training:write`（全员，自助发布的前提；只能编辑自己创建的）/ `training:admin`（管理所有、下架）。

### 前端

- `navigation.js` 加一条 entry：培训速递（`training:read`）；创建入口是列表页按钮，不单开菜单。
- `views/training/TrainingList.vue`：照 DictManagement + ExpoLeads 标杆（useListPage + utils/feedback）。
- `views/training/TrainingDetail.vue`：AI HOT 式排版；颜色走 tokens.css。
- `views/training/TrainingEditor.vue`：四步向导，state+方法全部拆 `composables/useTrainingEditor.js`（500 行红线）；上传用 `components/AppUpload`。
- 编辑器与详情页动工前加载 `emil-design-eng` skill（约定 13.5）。

### 里程碑（约 3 个工作日）

1. **M1**：迁移 + 后端四件套 + AI 起草管线（含 JSON schema 校验与测试）
2. **M2**:前端三页 + 强引导向导（含移动端走查）
3. **M3**：钉钉推送 + 有用/浏览计数 + check_conventions/pytest/build + 文档三处同步 + 对抗性审查

### DoD 附加项（本模块特有）

- AI 起草管线要有测试：给定固定材料文本，断言草稿 JSON 过 schema；
- 发布校验要有测试：★ 分区缺失时 publish 必须 4xx；
- 手机 375px 宽度下四步向导可完整走通（截图证据）。

## 五、明确不做（V1）

- 录音/视频自动转写（用钉钉 AI 听记过渡，需求真实出现 ≥3 次再评估内置）
- 审核流、版本历史、评论区
- 独立站点/免登录访问、RSS
- 培训日历/报名管理（那是另一个问题域）
