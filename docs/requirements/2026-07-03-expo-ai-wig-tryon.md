## 〇、品牌定位输入（所有设计决策的上位约束）

来自两份营销文档的核心结论，直接约束本应用的每一处文案与交互：

- **品牌**：莱莎健康假发（内贸），品类最贵（3000+），竞品是 2000-2800 的好产品——竞争不是"真 vs 假"而是"好 vs 更好"
- **双轨定位**：情感线「气质升级」先打动（想要）→ 理性线「久戴如新」再说服（值得）→ 偶尔点身份（尊贵感）。**顺序不能反**
- **时间维度差异化**："新的时候大家都好看，半年后只有莱莎还好看"——证据前置（6 个月对比实拍），不让客户等半年
- **向上锚定**：至臻系列（6000-8000）的存在让 3000 经典款从"最贵"变成"高端里的优选"
- **短发为主**：内贸客群以短发为主，质感体现在光泽/蓬松/发际线/头旋，不是梳子测试
- **话术禁用词**：便宜 / 划算 / 性价比 / 打折 / 薅羊毛——**任何界面文案与 AI 生成话术都不得出现**（拉低定位）
- **人群画像**：35-55 岁女性为主，需求三类：发量丰盈（头顶/发缝）、白发遮盖、造型变换；决策心理是"对自己好、不将就"

## 一、产品定位（先说本质）

这不是一个"试戴工具"，而是一台**展位获客机器**。它的价值链是：

```
吸引驻足（视觉冲击） → 换取名片（注册即留资） → 制造记忆点（AI 效果图）
  → 数据沉淀（照片+特征+反馈 进方舟） → 销售武器（AI 话术+跟进策略）
```

由此推出三条最高设计原则：

1. **每一秒等待都必须好看**——AI 生成有 20~40 秒延迟，这段时间的分析动画本身就是展位的橱窗表演，围观者看到的动画比效果图更重要；
2. **注册门槛压到最低**——留资是目的，但字段每多一个，转化掉一截。必填只留 3 项（姓名/公司/联系方式），其余选填；
3. **数据必须回流方舟**——展会结束后，这批客户照片、特征分析、试戴偏好、销售反馈是比效果图值钱得多的资产，直接喂给已有的客户机会台/客户画像体系。

## 二、核心用户旅程

### 现场角色与设备模式

| 角色 | 设备 | 模式 |
|------|------|------|
| 展位客户（35-55 岁女性消费者为主） | 展位自备 iPad/大屏手机 | 不登录、不装任何 App，在展位设备浏览器全屏模式下完成全流程 |
| 销售人员 | 同一设备 / 自己手机 | 方舟账号登录后进入「展会模式」；反馈录入在同一 H5 内完成 |
| 市场/管理员 | PC 方舟后台 | 发型库维护、话术卡库维护、线索总览、数据导出 |

**关键决策：以展位自备设备为主载体**。展位设备由销售登录一次、全天服务所有客户，客户零门槛上手。客户想带走效果图 → 结果页生成带短链的二维码（复用方舟短链服务），微信扫码即可保存并可一键转发朋友圈（品牌落地页，再赚一次曝光）；注册留了手机号，效果图链接也通过短信补发一次（离场后的第二触点）。

**展位设备运行方式（H5 kiosk 模式）**：iPad Safari 打开 `https://leshine.work/expo/` → 「添加到主屏幕」全屏运行（`manifest` 配 standalone），iPadOS「引导式访问」锁定单应用防止客户误退出。相机走 `getUserMedia`（HTTPS 已具备）。

### 客户主流程（目标：注册到出图 ≤ 90 秒）

```
① 待机吸引屏（金色粒子人脸线框 + slogan「戴上那一刻，状态就回来了」，无人操作自动播放）
② 快速注册（称呼/手机号必填，微信号选填 + 两道大图标选择题：
   最关心什么（发量丰盈/白发遮盖/造型变换）+ 偏爱风格（知性优雅/减龄轻盈/自然日常）
   + 拍照存储同意勾选）
③ 拍照（正脸引导框 + 光线检测提示，可重拍）
④ AI 分析动画（15~25 秒：扫描线 → 特征点浮现 → 特征标签逐个揭晓：
   脸型/肤色/气质类型/适配发长——只揭晓正面标签，这一步是表演；
   等待间隙轮播老客户证言「这不像假发，像我三年前的自己」——社会证明填充等待）
⑤ 发型甄选揭晓（3 款卡片翻出，附"为什么适合您"一句话；其中 1 款为至臻系列带金标）
⑥ 效果图生成（3 张并行，约 20~40 秒；进度动画 + 已完成的先展示）
⑦ 结果页 = 佩戴前后对比滑块（拖动分割线看"换头"瞬间——比单张效果图冲击强一个量级；
   每款可标记 心动/再看看；「久戴如新」证据条一键展开 6 个月对比实拍；
   扫码带走；"换一批"；「请顾问过来」按钮）
⑧ 销售接力（销售切到反馈面板：AI 双轨话术卡（情感线开场 + 理性线跟进）+
   仅内部可见的发况备注 + 意向 A/B/C/D + 备注提交）
```

**隐私呈现红线（贯穿全流程）**：分析屏当众只展示正面特征（脸型/肤色/气质/适配发长）。**发量、头皮状况的判断只写进 `analysis_json` 的内部字段，仅在销售面板与 PC 后台展示**——当众说"您头顶稀疏"是把客户从展位赶走的最快方式。

## 三、系统架构

```
H5 试戴应用（主站 SPA 内独立路由组 /expo/*，空白布局无主站导航）
   │  HTTPS（展位 4G/5G 随身路由，不依赖场馆 Wi-Fi）
   ▼
方舟后端 新领域模块 app/expo/          ← 标准领域模块四件套 + 子 service
   ├── 注册/会话/结果/反馈 API（/api/expo/*）
   ├── 匹配引擎（规则打分，纯函数）
   ├── AI 管线（复用 app/ai/ 的 Provider/Preset/调用日志三层）
   │     ├── preset: expo_face_analysis   （多模态分析 → 结构化 JSON）
   │     ├── preset: expo_wig_composite   （图像编辑模型 → 合成效果图）
   │     └── preset: expo_sales_strategy  （文本模型 + 知识库 → 策略话术）
   ├── 存储：uploads/expo/（客户照片、效果图）
   └── 数据表：ark_expo_*（6 张，见第五节）
   
PC 方舟后台（frontend/src/views/expo/）
   ├── 发型库管理（多角度照片 + 合成 prompt + 适配标签维护）
   └── 展会线索台（客户列表 / 会话回放 / 反馈汇总 / 导出）
```

**架构上刻意不做的**：不做独立部署（就是方舟的一个领域模块）、不做实时 WebSocket（轮询 2 秒足够）、不自研人脸检测（多模态大模型一次调用全包）、不训练模型（图像编辑模型 + 好 prompt 足够）。

## 四、AI 管线设计（技术核心）

### 4.1 管线一：面部特征分析（expo_face_analysis）

- **模型要求**：多模态视觉模型（平台已有接入经验：运单 OCR 用 step-3.7-flash；本场景对精度要求更高，建议配置 GPT-5 级或 Gemini 级视觉模型，通过 AiProvider 配置，不写死）
- **输入**：客户正脸照 + 结构化输出 prompt
- **输出 JSON**（存 `ark_expo_sessions.analysis_json`）：

```json
{
  "gender": "female",
  "age_range": "40-50",
  "face_shape": "oval|round|square|heart|long|diamond",
  "skin_tone": {"depth": "fair|light|medium|tan", "undertone": "cool|warm|neutral"},
  "temperament": "知性优雅|减龄轻盈|自然日常|端庄大气",
  "features": {"forehead": "...", "cheekbones": "...", "jawline": "...", "eyes": "..."},
  "suit_length": "short|bob|shoulder|long",
  "display_notes": "椭圆脸型，暖调白皙，知性优雅气质……",
  "internal": {
    "hair_condition": "发缝偏稀|头顶稀疏|发量正常|白发比例高",
    "sensitivity_hint": "肤质偏敏感迹象（如有）",
    "sales_note": "建议头顶补量款……"
  },
  "confidence": 0.92
}
```

**`internal` 字段是隐私红线的技术落点**：客户屏渲染只取顶层正面字段；`internal.*` 仅销售面板（登录态）与 PC 后台可见。气质类型（temperament）与注册页的风格偏好共同驱动匹配和话术。

- **已知坑直接规避**（cerebrum 记录在案）：推理模型 content 为空要 fallback reasoning 字段；AI 返回字段值需 `_clean` 后处理；prompt_snapshot 截断 base64。分析 preset 的 prompt 里给正反例并要求"只输出 JSON"。

### 4.2 管线二：发型匹配（规则引擎，不是 AI）

**刻意用规则打分而非让 AI 挑**——可解释（"为什么推这款"能讲给客户听）、可调（展会现场发现某款推得太少，改权重立即生效）、零延迟零成本。

```
score(wig, analysis, reg) =
    gender 硬过滤（不符直接排除）
  + face_shape 匹配        × 30    ← fit_tags.face_shapes 命中（圆脸→碎刘海款，方脸→长鬓角款，
                                      映射规则直接来自直播话术 v4 弹幕应对表）
  + skin_tone 匹配         × 20    ← 深浅 + 冷暖调分开算（发色与肤色同调）
  + need 需求匹配          × 25    ← 注册题"最关心什么"：发量丰盈→头顶补量款优先，
                                      白发遮盖→全头套自然色系，造型变换→款式多样性
  + temperament/style 偏好 × 15    ← AI 气质类型 + 注册风格偏好
  + age_range 重叠度       × 10
  + priority 人工权重       （平手时决胜；用于主推款加权）
→ 取 Top 3；"换一批" = 第 4~6 名
```

**至臻锚点规则（向上锚定策略的产品化）**：`ark_expo_wigs.series` 区分 classic/zhizhen。Top 3 组合规则——若至臻系列有款进入前 6 名，**强制占据 3 席之一**（带金色"至臻系列"标识）。目的不是卖至臻，是让客户先看过 6000+ 的存在，3000 经典款自然成为"高端里的优选"——营销策略 v2 的锚点效应直接编码进匹配引擎。

权重存 `config/expo_matching.yaml`（沿用平台 YAML 业务规则模式），现场可改。

### 4.3 管线三：效果图合成（expo_wig_composite）

- **模型选型**：图像编辑类模型（image editing / inpainting 能力）。平台在发色数字化模块已接入 **GPT-Image-2**（色板图生成 + ΔE 校验），管线直接复用该 Provider；同时配置一个备用图像模型（如 Gemini 图像编辑或 Seedream，经由中转站），AiProvider 后台秒切。**开发第一周就要做的事：拿 10 张真人照 × 5 款假发实测两家模型的合成质量与耗时，用数据定主备**——这是全项目最大的技术风险，必须最先验证。
- **输入组装**：客户照片 + 该假发的 1~2 张参考图（正面 + 斜侧） + 合成 prompt。每款假发的 prompt 存在发型库记录里（`composite_prompt` 字段），模板化：

```
Replace the person's hair with the wig shown in the reference images:
{wig_description}. Keep the face, skin tone, expression and background
完全一致。发际线过渡自然，头发落在肩部的物理关系真实，光照方向与
原照片一致。{per_wig_extra_prompt}
```

- **执行**：3 款并行（`run_in_executor` 线程池，复用 OCR 的自建 SessionLocal 模式），单张预算 20 秒、超时 60 秒；每张完成即推给前端（轮询到就展示，不等三张齐）。
- **失败兜底**：单张失败自动用第 4 名补位重试一次；全失败则展示该发型的模特实拍图 + 道歉文案（"现场网络拥堵，为您展示模特实拍效果"），体验不断链。

### 4.4 管线四：销售话术生成（expo_sales_strategy，双轨结构）

结果页出图的同时**异步**触发（客户看图的时间正好够生成）：

- **输入**：analysis_json（含 internal 发况）+ 注册信息（关心什么/风格偏好）+ 客户标记"心动"的款 + **话术卡库检索结果**（见第 5.1 节 ark_expo_scripts）
- **输出结构（双轨，直播话术 v4 的方法论产品化）**：
  1. **情感线开场**（先打动）：基于气质类型与试戴反应生成，模板源自"气质升级"话术族——"戴上的瞬间状态回来了"
  2. **理性线跟进**（再说服）：按客户特征自动选角度——长期佩戴→「久戴如新 12-18 个月」；发况敏感信号→「SGS 无甲醛 + 医用级亲肤网料」；犹豫价格→「一顶顶两顶」算账模板（3200 vs 5600）；心动了至臻款→顺势锚点话术
  3. **异议应对**：从弹幕 FAQ 库检索 2 条该客户最可能问的（如"能用多久"→"每天不到 7 块钱"）
- **禁用词硬校验（机器规则，不靠 prompt 自觉）**：生成结果过一遍禁用词过滤（便宜/划算/性价比/打折/薅羊毛），命中即带否定反馈重新生成一次，仍命中则该句替换为话术卡库的原文模板——**AI 负责个性化，规则负责底线**
- system_prompt 基座 = 品牌关键词矩阵（情感/理性/身份三层用词）+ 话术优先级（先情感→再功能→点身份，不许反）+ 禁用词表，全部来自直播话术 v4 文档，作为 preset 配置可随品牌策略版本更新

### 5.1 知识库建设：两份营销文档的结构化入库方案

营销策略 v2 与直播话术 v4 不是"存档文档"，拆开后分三个去处：

**① 话术卡库 `ark_expo_scripts`（~30 张卡，一次导入 + 后台可维护）**

| script_type | track | 来源内容 | audience_tags 示例 |
|------------|-------|---------|-------------------|
| opener 开场 | emotional | 气质变化钩子（"差在头发"）、身份认同钩子（"买的是稳稳的精致感"） | 通用 / 职业女性 |
| opener 开场 | rational | 健康关切钩子（"贴头皮 8 小时，看过成分吗"） | 敏感肌信号 |
| demo 演示引导 | rational | 光泽度对比 / 手感对比 / 发际线头旋特写 / 亲肤网料展示（短发五演示） | 短发客户，供销售现场配合实物 |
| closer 逼单 | identity | 身份定位法（"La Mer 与大宝"、"值得更好的"） | 高消费信号 / 心动至臻款 |
| closer 逼单 | rational | 状态投资法（每天省 30 分钟=一年 200 小时）、健康投资法、一顶顶两顶算账 | 犹豫价格 / 长期佩戴 / 敏感肌 |
| objection/faq | — | 弹幕应对 6 条（能用多久→每天不到 7 块钱；为什么贵→原料+时间；短发像男的→弧度层次…） | 按问题类型 |

每张卡带 `evidence_points`（可引用的硬证据：保鳞辫发不酸洗 / SGS 无甲醛无重金属 / 22 岁原生辫发 5 道筛选 / 1992 年起 33 年 · 164 国 · 3000+ 沙龙 / 12-18 个月佩戴寿命 / 7 天无理由退换）——AI 生成话术时只允许引用证据库里的事实，**不许自编数据**。

**② AiPreset system_prompt 基座（品牌语言宪法）**

关键词矩阵（情感层：气质/气场/状态/质感/精致/尊贵；理性层：健康/无化学/亲肤/持久/保鳞；身份层：懂的人选莱莎/品位/不将就）+ 话术优先级（先情感→再功能→点身份）+ 禁用词表。随品牌策略版本迭代（当前 v4），改 preset 即全线生效。

**③ 展位物料清单（非软件，但 H5 依赖）**

营销策略 P0 行动与 H5 的交集：6 个月对比实拍素材（喂给结果页"久戴如新"证据条 + evidence_refs）、10+ 老客户证言（喂给分析屏等待轮播）。**这两项物料 W1 就要向市场部提需求**，是 H5 内容完整度的外部依赖。

**数据回流**：展会后跑一个一次性脚本，把 intent_level ∈ {A,B} 的 expo 客户批量写入 `ark_customer_opportunities`（客户机会台），source 标记 expo，销售在既有工作台继续跟进——闭环不新造第二套跟进系统。

## 六、API 设计（/api/expo，标准领域模块）

| 端点 | 权限 | 说明 |
|------|------|------|
| `POST /register` | expo:write | 客户注册（含 consent），返回 customer_id |
| `POST /sessions` | expo:write | multipart 上传照片 → 建会话，异步跑分析 |
| `GET /sessions/{id}` | expo:write | 轮询：status + analysis + 匹配结果 + 已完成的效果图（**一个轮询端点覆盖全流程**，前端不用多接口拼装） |
| `POST /sessions/{id}/generate` | expo:write | 对匹配 Top3 触发合成；body 可传 wig_ids 指定（换一批） |
| `POST /results/{id}/reaction` | expo:write | 客户点喜欢/一般 |
| `GET /results/{id}/share` | 无鉴权（签名短链） | 客户扫码落地页（品牌页 + 效果图下载） |
| `POST /customers/{id}/feedback` | expo:write | 销售反馈录入 |
| `GET /customers/{id}/strategy` | expo:write | 销售策略话术（生成中返回 pending） |
| `GET /wigs` `POST /wigs` `PUT /wigs/{id}` | expo:read / expo:admin | 发型库 CRUD（PC 端） |
| `GET /leads` | expo:read | 线索列表（PC 端，筛选届次/意向/日期，导出 Excel） |

约定对齐（宪法红线）：全端点 `ok()` 信封、权限 Depends、新增 `expo:read/write/admin` 走 seed 三步、前端 `api/expo.js` 走 `createApiClient` 并在 clients.js 登记、PC 页面在 navigation.js 注册。

## 七、H5 前端设计

### 7.0 技术承载（H5 定案后的落地方式）

- **不建第二套代码库**（架构评估已把 `/m/` 平行实现列为教训）：expo 是主站 SPA 内的一个**独立路由组** `/expo/*`，用空白布局（无 MainLayout 导航），路由在 navigation.js 以 hidden entry 注册、组件放 `views/expo/`，API 走 `api/expo.js`（createApiClient + clients.js 登记）——与主站同构、同构建、同部署。
- **相机**：`navigator.mediaDevices.getUserMedia`，前置摄像头 + `<video>` 取景 + canvas 截帧。仅 HTTPS 可用（生产 leshine.work 已满足；本地开发用 `localhost` 豁免）。
- **全屏体验**：`manifest.json`（standalone + 曜石黑主题色）支持 iPad 添加到主屏幕；页面内兜底 `requestFullscreen`。
- **动画**：CSS 动画为主（原型即规格），粒子/扫描线均纯 CSS；echarts 等重依赖不进 expo 路由 chunk（配合架构评估 F-7 的 manualChunks）。
- **待机回归**：全局 idle 计时器 60 秒无触摸自动回待机屏并清除上一位客户的会话状态（隐私 + 下一位可用）。

### 7.1 视觉主题：「曜石 × 香槟金」

- **基调**：深曜石黑底（#0c0a08 系）+ 香槟金（#e8c479）单一强调色 + 细金线描边，大面积留黑，光感来自径向金辉与粒子，**不用紫色渐变、不用多彩**——奢华感来自克制
- **字体**：数字/英文用衬线体（Cormorant Garamond 风格，登记页大标题极具品牌感），中文正文 system 苹方/思源，加宽字距
- **动效三个高光时刻**（其余克制）：
  1. 待机屏：金色粒子聚成人脸轮廓线框，缓慢旋转呼吸——远处看是一件展品
  2. 分析屏：横向扫描线扫过照片 + 特征点星座式连线浮现 + 特征标签逐条揭晓（每条 stagger 400ms，制造"AI 正在读你"的仪式感）
  3. 结果揭晓：效果图从金色光晕中淡入 + 卡片视差翻转
- 移动布局已知坑直接规避：`box-sizing: border-box` 全局、flex 子元素 `min-width:0`（cerebrum 2026-06-08 条目）；iPad Safari 注意 `100vh` 含地址栏问题，用 `100dvh`

### 7.2 页面清单（7 页）

| 页面 | 要点 |
|------|------|
| attract 待机屏 | 无操作 60 秒自动回到此页并清上一位会话；粒子人脸线框 + 「戴上那一刻，状态就回来了」 |
| register 注册 | 称呼+手机号必填、微信号选填；两道大图标选择题（最关心什么 / 偏爱风格）+ consent；30 秒完成 |

| matching 甄选揭晓 | 3 张发型卡翻出 + "为什么适合您"一句话；至臻系列款带金标（锚点） |

### 7.3 PC 方舟后台（2 个新页面）

- `views/expo/WigLibrary.vue` — 发型库管理：列表复制 DictManagement 标杆 + useListPage；编辑抽屉含多角度图上传、fit_tags 标签选择器（选项走 sys_dict 字典：face_shape/skin_depth/undertone/style 四个 dict type，复用字典管理）、composite_prompt 编辑框 + 「试合成」按钮（选一张测试照即时验证 prompt 效果——**prompt 调优的核心工作台**）
- `views/expo/ExpoLeads.vue` — 展会线索台：届次筛选 + 客户列表（照片缩略、意向、反馈）+ 详情抽屉（会话回放：分析结果含内部发况 + 3 张效果图 + 心动标记）+ Excel 导出 + 「转入客户机会台」批量操作
- `views/expo/ScriptLibrary.vue` — 话术卡库管理：按 script_type/track 分组的卡片列表 + 编辑（内容/人群标签/证据点）+ 启停。市场部换话术版本不经过开发

### 7.4 从营销文档衍生的功能设计（本次吸收的拓展点）

1. **前后对比滑块**（已进结果页）：直播话术 v4 演示 3"同人换戴气质对比"的数字化——拖动分割线的"换头瞬间"就是话术里"整个气质出来了"的自助版
2. **久戴如新证据条**（已进结果页）：营销策略 v2"时间证据前置"的落点——效果图旁一键展开该款 6 个月对比实拍，把"半年后还好看"从话术变成客户自己翻到的证据
3. **至臻锚点进匹配规则**（已进 4.2）：向上锚定策略产品化，Top3 强制含至臻款
4. **等待期社会证明**（已进分析屏）：15~40 秒的等待动画间隙轮播老客户证言，替代干等
5. **禁用词硬校验**（已进 4.4）：品牌话术规范用规则兜底，不赌 AI 自觉
6. **数据反哺直播与选品**（展后价值，零额外开发）：试戴数据天然回答三个问题——哪款"心动率"最高（直播主推款依据）、什么脸型/气质的客户选什么款（话术库的"XX 脸选 XX 款"从经验变成数据）、至臻款的心动率（向上锚定策略是否成立的直接验证）。展后从 ark_expo_results/sessions 出一张统计报表即可
7. **到店复试闭环**（轻量）：销售面板 next_action 提供「约到店复试」快捷项，展会线索直通体验店——展位成交不了的高客单，靠到店第二触点

## 八、展会现场保障（非功能需求）

| 风险 | 对策 |
|------|------|
| 场馆网络不可用 | 展位 4G/5G 随身路由为主链路；H5 全流程只依赖 leshine.work 一个域名 |
| 图像模型现场抽风 | AiProvider 主备双配置，后台秒切；单张失败自动补位重试（4.3 节） |
| 生成排队（多客户同时） | 后端队列上限 3 并发，超出排队；前端显示"前方 N 位"；分析动画时长弹性拉伸吸收等待 |
| 彻底断网 | **演示模式**：预生成 5 组模特照全流程数据随构建打包（静态 JSON + 图片），一键切换纯本地播放，动画照跑——展台永远不冷场。H5 下配 Service Worker 预缓存 expo 路由资源，刷新页面也不依赖网络 |
| 客户照片隐私（PIPL） | 注册页 consent 明示（存储用途+保留 90 天）；分享落地页不含人脸原图 URL 猜测空间（签名短链）；后台一键删除；**uploads/expo 纳入每晚备份**（架构评估 S7 项的备份机制上线后自动覆盖） |
| 现场无人会调试 | 销售面板内置「自检」按钮：连通性 + AI provider 探活 + 最近一次生成耗时，红绿灯展示 |

**成本量级**（供拍板）：单客户 = 1 次视觉分析（约 ¥0.05~0.2）+ 3~6 张图像生成（约 ¥0.3~1.5/张）+ 1 次策略生成 ≈ **¥1.5~8/客户**；按展会 3 天 × 200 客户，AI 成本封顶约 ¥5000，相对展会获客成本可忽略，不做限流设计。
